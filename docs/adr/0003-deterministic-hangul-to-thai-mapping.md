# Deterministic jamo-to-Thai mapping for Korean Transliteration (supersedes ADR-0001)

ADR-0001 chose Cloudflare Workers AI (Qwen3) to generate Thai Transliterations. In practice the general-purpose model produced phoneme-level hallucinations that no amount of prompt/temperature tuning fixed reliably — e.g. 하게 (ha-ge) came out as "กั-คี" (ga-ki), because the model guesses pronunciations statistically rather than following Korean's fixed rules. Korean-to-phonetic conversion is fully deterministic: a Hangul syllable decomposes into onset/vowel/coda jamo via Unicode math, and pronunciation rules (including 연음 liaison, where a coda moves onto a following vowel-initial syllable, and compound-coda splitting) are well-defined. We replaced the AI call with a **deterministic jamo-to-Thai mapper** (`worker/transliterate.ts`): fixed onset/vowel/coda tables plus a liaison pass, with non-Hangul text (English, section markers, punctuation) passing through untouched. This eliminates the phoneme hallucinations, is instant (microseconds vs ~40s per song), removes the Workers AI binding and neuron-budget concern entirely, and needs no KV cache or retry/fallback logic since identical input always yields identical output. Trade-offs: Thai script cannot represent every Korean distinction (tense consonants ㄲ/ㄸ collapse onto their plain counterparts; ㅡ/ㅓ are approximated), and only liaison is modelled — more advanced sandhi (nasalization, tensification, ㄴ-insertion) is not — but for a singalong phonetic reading these approximations are acceptable and, crucially, consistent.
